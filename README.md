Проект для автоматизированного тестирования API

Описание задания:

Проект выполнен в рамках тестового задания и включает два обязательных этапа.
Все результаты выполнения размещены в одном репозитории, а все отчётные файлы находятся в корневом каталоге проекта.

Задание 1 - Составление баг-репортов

На основе предоставленного скриншота страницы карьерного сайта были выявлены баги, указаны их приоритеты и приведено обоснование выбранных приоритетов.

Описание файлов:
    - `BUGS-TASK-1.md` — список баг-репортов.

Задание 2.1 — API тесты (RestAssured)

API для тестирования: https://qa-internship.avito.com

Кейсы:
    1. Создать объявления (`POST /api/1/item`) с проверкой:
        - успешного создания с валидными данными;
        - обработки отсутствующих обязательных полей;
        - проверки типа данных полей;
        - обработки некорректного JSON.
    2. Получить объявления по его идентификатору (`GET /api/1/item/{id}`) с проверкой:
        - существующего `id`;
        - несуществующего `id`;
        - невалидного `id`.
    3. Получить все объявления по идентификатору продавца (`GET /api/1/{sellerID}/item`) с проверкой:
        - существующего продавца;
        - несуществующего продавца;
        - невалидного `sellerID`.
    4. Получить статистику по айтем id (`GET /api/1/statistic/{id}` и `GET /api/2/statistic/{id}`) с проверкой:
        - существующего `id`;
        - несуществующего `id`;
        - невалидного `id`.

Требования:
    - Java 17+.
    - JUnit 5 для написания тестов.
    - Maven для сборки проекта.
Все тесты реализованы в классе ItemApiTests.java

Описание файлов:
- `src/test/java/com/example/avito/ItemApiTests.java` — автотесты API.
- `TESTCASES.md` — список тест-кейсов.
- `BUGS.md` — список баг-репортов.
- `REPORT.md` — отчет с пояснениями и логикой тестирования.

Подготовка и запуск:

После клонирования репозитория выполните команду:

```bash
mvn clean install
```

Запуск автотестов Через Maven:

```bash
mvn clean test
```
