# Проект для автоматизированного тестирования API

## Описание задания

Проект выполнен в рамках тестового задания на позицию QA-тестировщика и включает **два обязательных этапа**.  
Все результаты выполнения размещены **в одном публичном репозитории**, а все отчётные файлы находятся **в корневом каталоге проекта**, согласно требованиям задания.

---

## Задание 1 — Составление баг-репортов

На основе предоставленного скриншота страницы карьерного сайта Авито были выявлены дефекты интерфейса.  
Для каждого бага указан приоритет (**High / Medium / Low**) и приведено обоснование выбранного уровня приоритета.

### Файлы:
- `BUGS-TASK-1.md` — список баг-репортов по Заданию 1.

## Задание 2.1 — Автоматизированное тестирование API (RestAssured)

**API для тестирования:**  
https://qa-internship.avito.com

В рамках задания были разработаны тест-кейсы и реализованы автоматизированные API-тесты для проверки работы микросервиса объявлений.

---

### Покрытые сценарии

#### 1. Создание объявления  
`POST /api/1/item`

Проверки:
- успешное создание объявления с валидными данными;
- обработка отсутствующих обязательных полей;
- валидация типов данных полей;
- обработка некорректного JSON.

#### 2. Получение объявления по идентификатору  
`GET /api/1/item/{id}`

Проверки:
- существующий `id`;
- несуществующий `id`;
- невалидный `id`.

#### 3. Получение объявлений по идентификатору продавца  
`GET /api/1/{sellerID}/item`

Проверки:
- существующий продавец;
- несуществующий продавец;
- невалидный `sellerID`.

#### 4. Получение статистики по объявлению  
`GET /api/1/statistic/{id}`  
`GET /api/2/statistic/{id}`

Проверки:
- существующий `id`;
- несуществующий `id`;
- невалидный `id`.

## Используемые технологии и требования

- **Java 17+**
- **JUnit 5** — тестовый фреймворк
- **RestAssured** — тестирование REST API
- **Maven** — сборка и управление зависимостями

Все автоматизированные тесты реализованы в одном классе:
- `ItemApiTests.java`

---

## Структура и описание файлов

- `src/test/java/com/example/avito/ItemApiTests.java` — автоматизированные API-тесты  
- `TESTCASES.md` — список тест-кейсов  
- `BUGS.md` — баг-репорты, выявленные в ходе тестирования API (Задание 2)  
- `REPORT.md` — отчет с пояснениями, допущениями и логикой тестирования

## Подготовка и запуск проекта

### Сборка проекта

После клонирования репозитория выполните команду:

```bash
mvn clean install
```

Запуск автотестов Через Maven:

```bash
mvn clean test
```
