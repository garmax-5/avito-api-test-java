## [ABUG-1] POST-запрос на /api/1/item с валидными данными - тело ответа не соответствует контракту

**Описание:**  
При выполнении POST-запроса на эндпоинт /api/1/item с валидными данными объявление успешно создаётся, однако тело ответа не соответствует контракту, описанному в postman_collection. Фактический ответ не содержит ожидаемого объекта, что делает невозможным использование возвращенного id в последующих запросах.

**Шаги воспроизведения:**
1. Отправить POST-запрос на https://qa-internship.avito.com/api/1/item
2. Проверить код состояния
3. Передать валидное тело запроса вида:
```json
{
  "sellerID": 111111,
  "name": "testItem",
  "price": 9900,
  "statistics": {
    "likes": 21,
    "viewCount": 11,
    "contacts": 43
  }
}
```

**Ожидаемый результат:**
1. Запрос успешно отправлен на сервер
2. HTTP Status: 200 OK
3. Создан первый объект объявления с id = <генерируется новое значение> Тело ответа в формате JSON возвращается от сервера и будет иметь следующий вид:
```text
{
  "id": <генерация нового значения типа string>,
  "sellerId": 111111,
  "name": "testItem",
  "price": 9900,
  "createdAt": <генерация нового значения типа string>,
  "statistics": {
    "likes": 21,
    "viewCount": 11,
    "contacts": 43
  }
}
```

**Фактический результат:**
1. Запрос успешно отправлен на сервер
2. HTTP Status: 200 OK
3. В ответе возвращается только поле status без данных объявления:
```text
{
  "status": "Сохранили объявление - <UUID>"
}
```

**Окружение:** браузер Chrome  
**Приоритет:** Blocker

## [ABUG-2] GET-запрос на /api/1/item/{id} с несуществующим идентификатором - тело ответа не соответствует контракту

**Описание:**  
При выполнении GET-запроса для получения объявления по несуществующему идентификатору сервис возвращает HTTP-status 404 Not Found, однако структура тела ответа не соответствует контракту, описанному в postman_collection.

**Шаги воспроизведения:**
1. Отправить GET-запрос на https://qa-internship.avito.com/api/1/item/{id}, где id - несуществующий идентификатор
2. Проверить код состояния
3. Проверить тело ответа сервера

**Ожидаемый результат:**
1. Запрос успешно отправлен на сервер
2. HTTP Status: 404 Not Found
3. Тело ответа в формате JSON возвращается от сервера и содержит информацию:
```text
{
  "result": "<string>",
  "status": "<string>"
}
```

**Фактический результат:**
1. Запрос успешно отправлен на сервер
2. HTTP Status: 404 Not Found
3. Тело ответа в формате JSON возвращается от сервера и содержит информацию:
```text
{
  "result": {
    "message": "<string>",
    "messages": {
      "<string>": "<string>"
    }
  },
  "status": "<string>"
}
```

**Окружение:** браузер Chrome  
**Приоритет:** Major
